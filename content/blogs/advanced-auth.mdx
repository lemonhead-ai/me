## Setting up the Database Schema
When dealing with enterprise applications, the default adapters are rarely enough. We need to define custom models for our users.

```prisma
model User {
  id            String    @id @default(cuid())
  name          String?
  email         String?   @unique
  emailVerified DateTime?
  image         String?
  role          Role      @default(USER)
}
```
